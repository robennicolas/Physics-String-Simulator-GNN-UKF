# Physics String Simulator — GNN + UKF

> Real-time 2D string simulation combining classical physics (Verlet integration), state estimation (Unscented Kalman Filter), and a learned dynamics model (Graph Neural Network).

---

## Overview

This project explores whether a **Graph Neural Network can replace a physics engine** to simulate the dynamics of a spring-mass string in real time. It also implements an **Unscented Kalman Filter** to estimate the system state from noisy observations.

The pipeline is fully end-to-end: data is generated by the physics simulator, fed into a PyTorch GNN, and the trained model is exported and deployed back into the C++ simulator via TorchScript.

## Graph Neural Network

The string is represented as a **graph**: each mass point is a node, each spring is an edge.

**Node features** `[x, y, vx, vy, is_fixed]` — position, velocity, boundary condition  
**Edge features** `[length, tension, dir_x, dir_y]` — geometric and mechanical state of each spring  
**Target** — acceleration `[ax, ay]` at the next timestep

The model uses an encode → message-passing → decode architecture:

```
Node encoder   : Linear(5  → 128)
Edge encoder   : Linear(4  → 128)

× 3 layers:
  message()    : MLP(384 → 128)   concat(x_i, x_j, edge_attr)
  update()     : MLP(256 → 128)   concat(x, aggregated_messages)
  residual connection at each layer

Decoder        : Linear(128 → 2)  → predicted acceleration
```

All features are standardized (zero mean, unit variance). Normalization constants are printed at dataset creation and copy-pasted into the C++ header for inference.

## Unscented Kalman Filter

A custom UKF is implemented to estimate the full system state `[x, y, old_x, old_y]` for all nodes from noisy observations. Key design choices:

- **Sigma points** computed via Cholesky decomposition with diagonal loading fallback for numerical stability
- **Process model** — each sigma point is propagated through the full Verlet integrator
- **Measurement model** — identity (direct state observation), with a commented-out alternative using relative distances and angles between nodes (more realistic but requires circular mean for angle handling)
- **Weights** parameterized by `kappa` following the standard UKF formulation

---

## Data Collection

Data is generated by running the simulator with random perturbations:

- 50 snapshots, each with a random tip position forcing
- 300 frames per snapshot, tip released after 30 frames to let the string oscillate freely
- Logged every 5 frames → ~3000 training samples
- Train / Val / Test split: 70% / 15% / 15%

---

## Known Limitation — Rollout Stability

The model is trained on **one-step predictions** from ground truth states. In closed-loop rollout, small prediction errors accumulate and push the system out of the training distribution, causing divergence after a few seconds.

This is a known challenge in learned simulators. The standard fix is **noise augmentation during training** (partially implemented via `inject_noise`) or **rollout training** (DAgger-style), which is the next planned improvement.

## Build & Run

```bash
# C++ — requires SFML, Eigen, LibTorch
mkdir build && cd build
cmake ..
make

# Run GNN simulation
./simulator   # calls system.runGNN()

# Python — train the model
pip install torch torch_geometric pandas
python python/train.py
python python/eval.py   # exports model_scripted.pt
```
